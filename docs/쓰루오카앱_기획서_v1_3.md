## §3. 데이터 구조 (TypeScript 스키마)

> 개발팀은 `src/data/tripData.ts` 단일 파일로 모든 콘텐츠를 관리한다.

```typescript
interface TripData {
  meta: TripMeta
  flights: Flight[]
  accommodation: Accommodation
  days: Day[]
  transport: TransportLeg[]
  tips: Tips
}

interface TripMeta {
  appTitle: string          // "Tsuruoka Excursion"
  travelers: string[]       // ["CHUN SUNGBUHM", "CHUN JIYONG"]
  startDate: string         // "2026-08-06"
  endDate: string           // "2026-08-09"
  region: string            // "쓰루오카·사카타"
}

interface Flight {
  id: string                // "outbound" | "return"
  flightNumber: string      // "KE2197"
  bookingRef: string        // 항공권 예약번호 (TODO: 확인 필요)
  terminal: string          // 출발 터미널 (TODO: 확인 필요)
  departure: { airport: string; code: string; datetime: string }
  arrival: { airport: string; code: string; datetime: string }
  seats: string[]           // ["33A", "33F"]
}

interface Accommodation {
  name: string              // "쇼나이 호텔 스이덴 테라스"
  nameJa: string            // "庄内ホテル スイデンテラス"
  address: string
  phone: string             // "+81 235-25-7424"
  checkIn: string           // "2026-08-06"
  checkOut: string          // "2026-08-09"
  nights: number            // 3
  mapQuery: string
  heroImage: string         // 앱 빌드에 포함된 호텔 이미지 경로
}

interface Day {
  day: number               // 1~4
  date: string              // "2026-08-06"
  dayOfWeek: string         // "목"
  theme: string
  events: Event[]
}

interface Event {
  id: string
  time: string
  title: string
  type: "transport" | "sightseeing" | "meal" | "activity" | "accommodation"
  description: string
  mapQuery?: string         // 있으면 카드 탭 시 지도 열기
  cost?: string             // "¥1,250" 또는 "무료". 1인 기준. 분담 표기 금지.
  duration?: string
  alert?: string
  guide?: EventGuide
}

interface EventGuide {
  summary: string
  description: string       // \n 줄바꿈 처리
  tips?: string
}

interface TransportLeg {
  id: string
  from: string
  to: string
  mode: "bus" | "train" | "taxi" | "flight" | "walk"
  modeLabel: string
  duration: string
  cost: string              // 1인 기준. 분담 표기 금지.
  note?: string
}

interface Tips {
  weather: string           // 오프라인 폴백용 정적 날씨 정보
  payment: string
  misc: string[]
}
```

## §4. 화면 정의

### 4-1. 홈 대시보드 (Home)

**목적**: "지금 뭐 하지?"에 즉시 답하는 화면.

**구성 원칙**: 스크롤 없이 한 화면에 꽉 찰 것. 3개 영역만.

**영역 구성**:

1. **히어로 이미지**
   - 전체 너비. 현재 날짜 기준 호텔 이미지 표시.
   - 이미지는 빌드에 포함 (로열티 프리, 스이덴 테라스 논 위 풍경).

2. **날씨**
   - Open-Meteo 실시간. **1시간 간격 갱신**.
   - 오늘·내일 2일치를 좌우로 배치.
   - 표시 항목: 날씨 상태(맑음/흐림/비) + 최저·최고기온만.
   - 좌표: 38.73, 139.82 (쓰루오카)
   - 오프라인 시: 캐시 → tips.weather 정적 폴백.

3. **다음 일정**
   - 현재/다음 이벤트 1~2개. 시간·장소·아이콘.
   - 탭 → 해당 일자 타임라인으로 이동.

**제거된 항목**: 숙소 요약 카드, 앱 제목·기간·인원 요약 (더보기로 이동 또는 삭제)

---

### 4-2. 일자별 상세 일정 (Timeline)

**목적**: 각 날짜의 시간순 활동을 타임라인으로 제공.

**포함 정보** (DAY 1~4 공통):
- 날짜·요일·테마
- 시간별 이벤트 카드: 시각, 장소명, 설명, 아이콘
- 이동 구간 카드: 교통수단·소요시간·요금(1인)
- ⚠ 주의 항목 강조 표시

**주요 인터랙션**:
- DAY 탭 전환 (상단 세그먼트: DAY 1 / 2 / 3 / 4)
- **mapQuery 있는 이벤트 카드 탭 → Google Maps 열기** (`https://maps.google.com/?q=` + mapQuery)
- guide 있는 이벤트 카드 탭 → 바텀시트 슬라이드업
- ⚠ 항목: 카드 좌측 error 컬러 보더 + 경고 텍스트 박스

---

### 4-3. 예약 (Bookings)

**목적**: 예약 정보를 즉시 확인·복사할 수 있는 화면.

**구성 (이 3개만)**:

*1. 비행기 카드 (가는 편·오는 편 각 1장)*
- 편명, 예약번호, 터미널, 출발·도착 시간, 좌석

*2. 숙박 카드*
- 호텔 이름
- 전화번호 (탭 → tel: 링크)
- 주소

*3. 알 켓차노 카드*
- 현행 유지. 예약 미확정 상태 표시 (점선 테두리).
- 예약 확정 후 예약번호·날짜·시간 업데이트.

**주요 인터랙션**:
- 예약번호 탭 → 클립보드 복사 + 토스트 "복사됨"
- 전화번호 탭 → tel: 링크

2. `navigator.clipboard.writeText()` 호출
3. 토스트: "복사됨 ✓" (1.5초 후 자동 소멸)
4. 복사 아이콘 순간 체크 아이콘 전환 후 복귀

**탭 영역**: 최소 48×48dp

### 7-2. 일정 DAY 전환

**형태**: 수평 세그먼트 컨트롤 (DAY 1 | DAY 2 | DAY 3 | DAY 4)

- 활성 DAY: primary-container 배경 + 흰색 텍스트
- 비활성 DAY: surface-container 배경 + on-surface-variant 텍스트
- 오늘 날짜 기준 자동 선택. 여행 전/후에는 DAY 1 기본.

### 7-3. 지도 열기

**트리거**: mapQuery 있는 이벤트 카드 탭

**동작**: `window.open('https://maps.google.com/?q=' + encodeURIComponent(mapQuery), '_blank')`

mapQuery 없는 카드는 탭해도 지도 미동작.

### 7-4. 이벤트 안내 바텀시트

**트리거**: guide 필드가 있는 이벤트 카드 탭

| 영역 | 내용 |
|------|------|
| 핸들 | 드래그 핸들 바 |
| 헤더 | 이벤트 title + guide.summary |
| 본문 | guide.description |
| 하단 | guide.tips (있을 경우만) |

**닫기**: 아래 스와이프 / 외부 탭 / X 버튼. 최대 높이 85%.

### 7-5. ⚠ 주의 항목 강조

- 카드 좌측 보더: error 컬러 (#ba1a1a 계열)
- 경고 텍스트 카드 하단 별도 박스
