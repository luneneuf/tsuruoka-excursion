## 3. 데이터 구조

> **v1.5 변경 사항**:
> - `days[].events[]`에 `guide` 객체 필드 추가. guide가 있는 이벤트는 바텀시트 안내 콘텐츠 표시.
> - **`highlight` 필드 폐기**. 이벤트 카드 배경색은 `guide` 유무 + `type`으로 결정 (→ 5-1 참조). 기존 highlight 필드는 데이터에서 제거.

> **데이터 위치**: `flights` · `accommodations` · `activities` · `japaneseRequests` · `days` · `transport` · `tips` → `src/data/` 참조. 이 문서에는 v1.5 신규 추가된 `guide` 필드 스키마와 콘텐츠(11건)만 기록.

### 3-1. guide 필드 스키마

`days[].events[]` 중 안내가 필요한 이벤트에만 선택적으로 포함:

```typescript
guide?: {
  summary: string      // 한 줄 요약 — 바텀시트 헤더 서브텍스트
  description: string  // 본문 — 줄바꿈 \n 처리
  tips?: string        // 팁 — 있을 경우에만, surface-container-low 박스로 표시
}
```

---

### 3-2. guide 콘텐츠 (11건)

guide 없는 이벤트는 `src/data/`에 기록되며 기획서에 중복 기재하지 않음.

**① 나가마치 무사 저택 터** (11:00 · sightseeing)

summary: 에도 시대 무사 주거 구역, 흙담과 돌판길 미로

description: 나가마치(長町)는 에도 시대 가가 번의 중급·하급 무사들이 거주하던 주거 구역입니다. 지금도 흙담(土塀)과 돌판길이 좁은 골목 사이에 미로처럼 남아 있어 당시의 분위기를 생생하게 전합니다.

현재 주택가와 혼재해 있어 실제로 사람이 사는 골목 사이를 걷는 특별한 경험을 할 수 있습니다. 대표 명소인 노무라 저택(野村家)은 내부를 공개하고 있으며, 서원조(書院造) 양식의 방과 아담한 일본 정원이 잘 보존되어 있습니다.

겨울에는 흙담을 서리로부터 보호하는 전통 방식인 코모가케(コモ掛け, 짚 덮기)가 행해져 독특한 계절 풍경을 만들어냅니다.

tips: 노무라 저택 입장 추천(550엔, 약 30분 소요). 골목이 좁고 구조가 복잡해 지도보다 발길 닿는 대로 걷는 것이 오히려 즐겁습니다. 편한 신발 필수.

일본해(日本海)에서 잡아 올린 해산물이 주력입니다. 봄에는 흰새우(白エビ)와 아마에비(甘エビ, 단새우)가 제철이며, 겨울에는 즈와이가니(ズワイガニ, 대게류)와 방어가 인기입니다. 시장 안팎에 회 덮밥·해산물 덮밥 전문점이 여럿 있어 현지인과 관광객 모두에게 점심 명소로 자리잡고 있습니다.

**③ 겐로쿠엔** (14:30 · sightseeing)

summary: 일본 3대 정원, 마에다 번주가 180년에 걸쳐 조성

description: 겐로쿠엔(兼六園)은 가가 번(加賀藩)의 역대 번주 마에다 씨가 17세기부터 약 180년에 걸쳐 조성한 회유식 정원입니다. 고라쿠엔(岡山)·가이라쿠엔(水戸)과 함께 일본 3대 정원(日本三名園)으로 꼽히며, 국가 특별 명승지로 지정되어 있습니다.

'겐로쿠(兼六)'라는 이름은 중국 원림론에서 따온 말로, 광(広·넓음)·유(幽·고요함)·인(人·인공미)·창(蒼·고풍스러움)·수(Water·수경)·조(眺·전망) 여섯 가지 조건을 모두 갖췄다는 뜻입니다.

정원 내 코토지 등롱(琴柱灯籠)은 다리가 두 개인 특이한 석등으로, 가나자와를 상징하는 가장 유명한 피사체입니다. 4월에는 약 420그루의 벚꽃이 만개해 절경을 이룹니다.

tips: 입장료: 성인 320엔, 6~18세 어린이 100엔. 겐로쿠엔과 바로 연결된 가나자와성 공원을 함께 관람하면 효율적입니다. 코토지 등롱은 입구에서 오른쪽 연못 방향으로 도보 5분 거리입니다. 벚꽃 시즌(4월)에는 개장 직후(오전 7시)가 가장 한산합니다.

summary: 가가 100만 석 마에다 가문의 거성

description: 가나자와성(金沢城)은 1583년 초대 번주 마에다 도시이에(前田利家)가 입성한 이후 약 300년간 마에다 가문의 거성으로 기능한 성입니다. 가가 번은 에도 시대 도쿠가와 막부 직할령을 제외하면 일본 최대인 102만 석 규모의 번으로, 그 위세에 걸맞은 대규모 성이었습니다.

현존하는 히시야구라(菱櫓)·고산마이루(五十間長屋)·하시즈메몬 속문(橋爪門続櫓)은 2001년에 전통 기법으로 복원한 건물입니다. 성 내부는 공원으로 조성되어 있어 자유롭게 산책할 수 있으며, 겐로쿠엔과 이시카와몬(石川門)을 통해 직접 연결됩니다.

tips: 공원 입장 무료. 복원 건물 내부 관람은 별도 유료(300엔). 겐로쿠엔과 연결 출입구(이시카와몬 방향)를 이용하면 두 곳을 이어서 관람할 수 있습니다. 니노마루 광장은 탁 트인 공간으로 사진 찍기 좋습니다.

## 4. 인터랙션 정의

### 4-1. 클립보드 복사

**대상 항목**:
- 예약번호 3건 (숙소 2 + 체험 1)
- 일본어 요청 문구 1건

**동작**:
1. 사용자가 예약번호 영역 또는 복사 아이콘 탭
2. `navigator.clipboard.writeText()` 호출
3. 토스트 알림 표시: "복사됨 ✓" (1.5초 후 자동 사라짐)
4. 복사 아이콘이 순간적으로 체크 아이콘(done)으로 변경 후 복귀

**탭 영역**: 예약번호 텍스트 + 복사 아이콘을 포함하는 카드 전체 영역 (터치 타겟 최소 48×48dp)

### 4-2. 일정 DAY 전환

**위치**: 일자별 상세 일정 화면 상단

**형태**: 수평 세그먼트 컨트롤 (DAY 1 | DAY 2 | DAY 3 | DAY 4)

**동작**:
- 탭하면 해당 DAY 타임라인으로 전환
- 활성 DAY: primary-container 배경 + 흰색 텍스트
- 비활성 DAY: surface-container 배경 + on-surface-variant 텍스트

### 4-3. ⚠ 알림 항목 강조

**대상**: alert 필드가 있는 이벤트

**표현**:
- 카드 좌측 보더에 error 컬러(#ba1a1a) 또는 secondary-container 컬러 적용
- 알림 텍스트를 카드 하단에 별도 박스로 표시
- alert에 `alertRef`가 있으면 해당 일본어 요청 문구로 바로 이동하는 링크 포함

### 4-4. 오프라인 상태 표시

**동작**:
- `navigator.onLine` 감지
- 오프라인 전환 시: 화면 상단에 얇은 배너 표시 "오프라인 모드 · 모든 정보 이용 가능"
- 온라인 복귀 시: 배너 자동 사라짐 (0.5초 페이드)
- 컬러: surface-container-high 배경 + on-surface-variant 텍스트 (비위협적 톤)

### 4-5. 이벤트 안내 바텀시트

**트리거**: `guide` 필드가 있는 이벤트 카드 탭

**열기 동작**:
1. 사용자가 guide가 있는 이벤트 카드 탭
2. 바텀시트 슬라이드 업 애니메이션으로 올라옴
3. 배경(스크림)에 반투명 오버레이 적용

**바텀시트 구성**:

| 영역 | 내용 | 비고 |
|------|------|------|
| 핸들 | 드래그 핸들 바 (중앙 상단) | 닫기 제스처 힌트 |
| 헤더 | 이벤트 `title` + `guide.summary` | title: headline 폰트, summary: on-surface-variant |
| 본문 | `guide.description` | body 폰트, 줄바꿈(`\n`) 처리 |
| 하단 | `guide.tips` (있을 경우에만) | surface-container-low 배경 박스, 아이콘: lightbulb |

**닫기 동작**:
- 핸들 영역 아래로 스와이프 → 닫힘
- 바텀시트 외부(스크림) 탭 → 닫힘
- 닫기 버튼(X 아이콘) 제공 — 우상단

**탭 불가 이벤트**:
- `guide` 필드가 없는 이벤트 카드는 탭 반응 없음
- 커서 변화·리플 없음
- 배경색 차이(컬러 카드 vs. 기본 배경)로 자연스럽게 구분

**디자인 사양**:
- 모서리: `border-radius-3xl` (1.5rem) — 상단 좌우
- 배경: `surface-container-lowest` (#ffffff)
- 최대 높이: 화면의 85%

- 그림자: `0 -12px 40px rgba(27,29,14,0.10)`

### 가용 콘텐츠

**전부**. 이 앱의 모든 콘텐츠는 정적 데이터이므로 오프라인에서 100% 열람 가능해야 한다.

구체적으로:
- 4개 화면 전체
- 일정 데이터 (DAY 1~4)
- 예약번호·일본어 요청 문구
- 교통·비용 정보
- 참고 사항 (날씨·결제 팁)
- 클립보드 복사 기능
- 이벤트 안내 바텀시트 (guide 콘텐츠 포함 — 정적 데이터이므로 오프라인 완전 지원)

### 캐싱 전략 방향

- Service Worker를 통한 전체 앱 셸 + 데이터 프리캐싱
- 데이터가 정적이므로 Cache First 전략 적용
- 이미지 에셋도 빌드 시 번들에 포함하거나 설치 시 캐싱
- 외부 API 호출 1건 예외: Open-Meteo 날씨 API
  - 오프라인 시 처리 순서: 캐시(6시간) → tips.weather 정적 데이터 폴백
  - 날씨 API를 제외한 모든 콘텐츠는 네트워크 의존도 0

### 확정: 이미지 에셋

stitch 시안의 이미지는 `lh3.googleusercontent.com/aida-public/` 경로의 AI 생성 임시 URL이므로 만료됨.

**방침**: 아이콘 + 컬러 카드를 기본으로 하되, 홈 히어로 영역에 한해 로열티 프리 사진 1~2장을 빌드에 포함.
